## Screenshot Capture Guidelines

### Technical Specifications
- **Resolution:** 1920x1080 minimum
- **Format:** PNG for transparency support
- **Quality:** High resolution, no compression artifacts
- **Browser:** Chrome or Firefox for consistency

### Styling Guidelines
- **Consistent Lighting:** Use same cosmic background theme
- **Clean UI:** Hide browser chrome, focus on application
- **Readable Text:** Ensure all text is legible at presentation size
- **Color Consistency:** Maintain brand color palette

### Capture Tools
- **Browser DevTools:** For responsive views
- **Full Page Screenshots:** For complete oracle responses
- **Screen Recording:** For animations (convert to GIF)
- **Design Tools:** Figma/Sketch for diagrams

## Presentation Integration

### Slide Placement Strategy
- **Hero Images:** Full background with text overlay
- **Feature Screenshots:** Centered with callout annotations
- **Technical Diagrams:** Clean backgrounds, professional styling
- **UI Screenshots:** Show real usage, not staged demos

### Animation Considerations
- **Static Images:** For most slides
- **Subtle GIFs:** For loading states or key interactions
- **Video Clips:** For full demo sections only
- **Transition Effects:** Consistent with mystical theme

## Brand Consistency

### Visual Elements to Include
- **Cosmic Purple Gradients:** #9B59B6 primary color
- **Floating Particles:** Mystical atmosphere
- **Glassmorphism Effects:** Translucent cards
- **Crystal Ball Icon:** Brand recognition
- **Consistent Typography:** Inter font family

### Elements to Avoid
- **Generic Stock Photos:** Use actual application screenshots
- **Inconsistent Colors:** Stick to brand palette
- **Low Quality Images:** Maintain professional standard
- **Cluttered Layouts:** Keep focus on key features
